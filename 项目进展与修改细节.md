# 大模型驱动的交互式推理链增强关键技术研究与系统实现

## 本轮已解决的问题

原文档中列出的 5 个主要问题已完成工程化处理，系统从“基础原型”推进为具备可解释检索、等价辅助判断、元数据管理和量化评测能力的版本。

### 1. 公式等价判断能力不足

已新增 `EquivalenceService`，在原有字符串归一化和结构特征匹配之外，加入基于 SymPy 的最佳努力等价判断。

实现内容：

- 将常见 LaTeX 片段转换为可计算表达式；
- 支持等式两侧差值化简；
- 对无法完全符号化的复杂公式，回退到规范化骨架相似度；
- 搜索结果新增 `equivalence_score` 和 `equivalence_reason`，用于解释候选公式是否可能等价。

修改文件：

- `backend/app/services/equivalence_service.py`
- `backend/app/services/search_service.py`
- `backend/app/schemas/formula.py`
- `frontend/src/services/api.ts`
- `frontend/src/components/ResultCard.tsx`

### 2. 公式元数据不完整

已为公式数据模型和公式库补充难度、证明思路、应用场景和来源字段。

实现内容：

- `FormulaBase` 新增 `difficulty`、`proof_sketch`、`application_scenarios`、`source`；
- 148 条公式数据均已补齐上述字段；
- 后端加载旧数据时会自动推断缺失元数据；
- 新增和更新公式时支持保存这些元数据字段。

修改文件：

- `backend/app/schemas/formula.py`
- `backend/app/api/routes.py`
- `backend/app/data/formulas.json`
- `frontend/src/services/api.ts`

### 3. 语义检索缺少向量化工程接口

已新增 `VectorService`，当前使用本地确定性向量作为可运行方案，同时保留 pgvector 工程化边界。

实现内容：

- 基于 token 哈希生成稳定向量；
- 搜索排序新增 `vector_score`；
- 后端新增 `/admin/vector/status`，用于检查 `DATABASE_URL` 或 `PGVECTOR_DATABASE_URL` 以及 `psycopg2` 驱动是否就绪；
- 当前未配置数据库时自动使用本地向量方案，不影响系统运行。

修改文件：

- `backend/app/services/vector_service.py`
- `backend/app/services/search_service.py`
- `backend/app/api/routes.py`
- `frontend/src/components/ResultCard.tsx`

### 4. 外部大模型接口仍停留在预留阶段

已将 `_call_ai_api` 从占位函数改为 OpenAI 兼容 Chat Completions 调用。

实现内容：

- 支持 `OPENAI_API_KEY` 或 `LLM_API_KEY`；
- 支持 `OPENAI_BASE_URL` 和 `OPENAI_MODEL`；
- 返回内容按 JSON 解析后合并到公式分析结果；
- 外部 API 不可用或未配置时，自动回退到本地规则推理链。

修改文件：

- `backend/app/services/ai_service.py`
- `backend/app/api/routes.py`

### 5. 缺少检索效果评测集

已新增内置评测集和评测服务。

实现内容：

- 新增 `evaluation_queries.json`，覆盖二项式求和、Catalan 闭式、Stirling 与 Bell、中心二项式生成函数、范德蒙德卷积和 Burnside 引理；
- 新增 `EvaluationService`，计算 Recall@K、MRR、NDCG@K；
- 新增 `/admin/evaluation/run` 接口，可直接运行当前检索算法评测。

修改文件：

- `backend/app/data/evaluation_queries.json`
- `backend/app/services/evaluation_service.py`
- `backend/app/api/routes.py`
