# AGENT.md

## 项目名称

**组合数学公式检索系统**
暂定名：**CombFind / CombiOEIS / FormulaAtlas**

---

## 1. 项目概述

这是一个面向**组合数学公式检索**的垂直工具，目标类似 “**组合数学公式版 OEIS**”。

用户输入一个组合数学公式后，系统应完成以下核心任务：

1. 解析公式输入
2. 进行标准化处理
3. 提取结构特征
4. 检索知识库中最相近的已知公式
5. 返回相似度结果、匹配原因、公式详情与参考来源

本项目是一个**公式检索系统 / 知识库系统**，不是普通 CRUD 应用，也不是通用数学计算器。

## 2. 产品边界

Agent 在开发和实现时，必须始终遵守以下产品边界。

### 2.1 本项目是什么

这是一个：

- 面向组合数学公式的检索工具
- 结构化公式知识库
- 支持 AI 辅助分析的数学检索系统
- 强调“输入公式 -> 找相似公式 -> 给解释”的轻量产品

### 2.2 本项目不是什么

这不是：

- 通用符号计算系统
- 自动数学证明系统
- 全学科公式搜索引擎
- 论文写作平台
- 大型社区内容产品
- 单纯依赖 LLM 的聊天问答应用

### 2.3 MVP 绝对不做的事情

Agent 不应擅自扩展以下能力，除非明确要求：

- 自动严格证明
- 复杂 CAS 级别恒等变换
- OCR 图片识别
- 社区评论、点赞、关注等社交功能
- 多领域数学泛化支持
- 复杂推荐流
- 大规模协作编辑系统

## 3. 核心开发目标

### 3.1 第一优先级

**输入公式 -> 解析 -> 标准化 -> 检索 -> 返回结果**

### 3.2 第二优先级

## 7. 开发原则

## 7.1 先核心，后增强

Agent 必须优先实现：

1. 公式输入
2. 公式预览
3. 解析
4. 标准化
5. 相似检索
6. 结果展示
7. 管理后台录入

不要优先实现：

- 动画优化
- 复杂主题系统
- 多角色权限细分
- 非核心运营功能

## 7.2 结果可解释优先于“看起来聪明”

对于数学检索场景：

- 黑盒答案不可接受
- 相似结果必须尽量说明原因
- AI 输出只能作为辅助，不可伪装成严格数学证明

## 7.3 规则优先于幻觉

在涉及公式解析、标准化、分类、检索时：

- 优先使用确定性规则
- 优先使用结构化特征
- 再用 embedding 和 LLM 补充
- 不允许直接把 LLM 输出当最终数学事实

## 7.4 轻量优先

- 不过度工程化
- 不提前抽象未来十个版本的能力
- 所有设计围绕 MVP 验证价值展开

## 10. 后端实现指令

API 要清晰、稳定、面向前端使用，不要暴露内部复杂性。

- 公式原文
- 输入类型（latex / text）

输出：

- 是否解析成功
- 标准化结果
- 变量识别结果

- 基础检查提示

#### `POST /api/search`

输入：

- 公式原文

输出：

- 解析摘要
- 搜索结果列表

输出：

- 公式详情

- `POST /api/admin/formulas`

## 11. 公式处理实现原则

这是本项目最关键部分，Agent 必须重点遵守。

## 11.1 解析原则

支持输入：

- LaTeX
- 纯文本数学表达式

优先识别：

- 等式
- 求和
- 乘积
- 递推式
- 生成函数
- 组合函数符号

## 11.2 标准化原则

必须尽量做以下标准化：

- 变量名归一化
- 空格和括号冗余消除
- LaTeX 常见宏差异消除
- 求和上下界形式统一
- 左右表达式结构统一
- 常见组合对象别名统一

但要注意：

- 不要假装已经解决了“所有数学等价变形”
- 不要在没有可靠规则时强行化简
- 标准化结果要可追踪、可解释

## 11.3 允许的数学智能程度

可以做：

- 结构识别
- 变量重命名归一化
- 常见组合符号模式识别
- 基础一致性提示

不要默认做：

- 严格等价证明
- 深层恒等变换
- 完整数学真伪判定

---

## 12. 检索实现原则

## 12.1 检索架构

必须优先采用混合检索：

1. 精确 / 近似标准化匹配
2. AST / 结构特征匹配
3. 标签 / 分类召回
4. embedding 向量召回
5. 重排序

## 12.2 排序依据

排序应综合考虑：

- 标准化后的结构一致性
- 公式对象相似性
- 运算结构相似性
- 分类和标签相近性
- embedding 语义相似度

## 12.3 结果解释要求

每条结果至少给出一条匹配原因，例如：

- 共享相同求和结构
- 都包含二项式系数
- 变量归一化后结构高度一致
- 当前结果可能是输入公式的特例
- 差异主要在求和上界或常数项

## 13. AI 使用规范

## 13.1 AI 的角色

AI 只用于以下辅助任务：

- 公式类型识别
- 检查提示生成
- 匹配原因生成
- 候选结果重排序辅助

## 13.2 AI 不能直接承担的角色

AI 不能作为：

- 唯一检索引擎
- 严格证明器
- 唯一真值判断器
- 后台数据审核的唯一依据

## 13.3 AI 输出约束

所有 AI 生成内容必须符合以下要求：

- 明确区分“确定”和“推测”
- 不夸大结论
- 不输出“已证明”“完全等价”等未经规则验证的说法
- 优先使用“可能”“建议”“疑似”“结构上接近”等表达
